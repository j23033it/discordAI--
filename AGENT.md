# エージェント運用ルール（このプロジェクト）

このファイルは、本プロジェクトで作業するエージェント向けのシステム指示です。

## 必ず守る事項
1. 機能追加・修正の作業が完了したら、必ずコミットすること。
2. コミットメッセージは必ず日本語で記載すること。
3. 作業完了時にユーザーへ次アクションを提案する際は、`docs/github-actions-guide.md` を参照し、目的に応じて実行すべき GitHub Actions ワークフローを具体的に提案すること。

## ワークフロー提案の方針
- 新着取得・通常通知の確認が目的: `AI Updates Polling`
- 日次集約通知の確認が目的: `AI Updates Daily Digest`
- UI表示や文面確認が目的（新着がなくても確認したい）: `AI Updates Preview Notification`

## 出力時の注意
- ユーザーへの案内は、実行するワークフロー名を明記すること。
- エラー調査時は、どのステップのログを確認するかも合わせて示すこと。

## 追記候補（採用はユーザー判断）
- コミット前に最低限のチェック（例: `python3 -m compileall -q src`）を実行するルール
- 1機能1コミットを原則にするルール
- 破壊的操作（履歴改変・強制push）前に必ずユーザー確認を取るルール
- 機密情報（APIキー等）を検知したら即時マスク・履歴対応を優先するルール
